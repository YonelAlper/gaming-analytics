Methods

Database design principles, data generation strategy, and analytical approach.

Database Design

The database was designed using a relational model in PostgreSQL. Primary and foreign key constraints were applied to ensure data integrity.

Data Generation

Synthetic (randomly generated) data were used instead of real player data. This approach avoids privacy concerns while enabling realistic analytical scenarios.

Analysis Strategy

  • Aggregations using SUM, AVG, and COUNT
  • Grouping and filtering across multiple dimensions
  • Comparative analysis (e.g., buyers vs non-buyers)
  • Temporal analysis based on session timestamps
